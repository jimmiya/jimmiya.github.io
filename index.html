Vulkan Renderer

I coded a renderer with C++ using the Vulkan API (with Vulkan 1.3 features: dynamic rendering!) as part of coursework at University.

The renderer uses PBR shading, alpha masking (for the trees and bushes), as well as multiple debug visualisations such as mipmap levels, depth and partial derivatives. It also has a pixelated / "mosaic" post-processing effect.

Physics Engine for the KillSwitch game engine

In the development of our custom KillSwitch game engine as part of coursework at University, I developed a full physics engine from scratch.

It allows for multiple types of rigid bodies and collider shapes. It contains a coarse AABB check and a narrower check with full implementations of the GJK+EPA algorithms for collision detection and contact generation.

Also an interpenetration and impulse-resolution (with friction) solver, queries, events, and more.
Extra logic was added for characters to be able to go up and down slopes and stairs, as well as to climb obstacles.

For more detailed information refer to the source code or request the double page academic report, which explains all implementation details.

Path tracing Renderer

A full-featured CPU Monte-carlo path tracer as part of coursework at univeristy

The renderer was coded in C++ and has a rasterized side-by-side comparison scene using OpenGL.

The raytracer uses physically correct phong shading, and has the following features:

  • Point and area lights with hard and soft shadows
  • Reflection, refraction (with beer's law) and the fresnel effect
  • Monte-carlo sampling
  • Next Event Estimation for indirect lighting
  • Anti-aliasing
  • Caustics

Mesh Processing Toolkit

I implemented a mesh processing toolkit in C++ as part of coursework at University.

Detailed set of features:

  • Conversion from triangle soup to indexed face
  • Conversion from indexed face to directed edge
  • Manifold testing for meshes
  • Topological analysis using Euler's formula
  • Simple Mesh repair
  • Mesh simplification using curvature analysis

Animation and Physics Simulation

Using an OpenGL renderer provided for the coursework, I added skeletal animation with animation blending, and a basic mesh-agnostic rigid-body physics engine, as part of coursework at univeristy.

Skeletal data and animation was processed using BHV format. I optionally added animation blending and position interpolation to make it cleaner.

Physical simulation features sphere and dodecahedra (processed using vertices, so basically shape-agnostic) colliding against terrain. Also added sphere to sphere collisions.

HellMist, a game made in 2 days using our custom game engine

As part of the Group Project coursework, where we made a custom engine from scratch, we also made a game to show and submit to the Game Republic Student Showcase!
